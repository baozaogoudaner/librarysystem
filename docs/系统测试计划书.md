# 图书馆智能服务系统 — 系统测试计划书

## 1. 测试概述

### 1.1 测试目标
- 验证系统功能完整性与正确性
- 确认微服务间接口交互正常
- 评估系统在高并发场景下的性能表现
- 保障系统安全性、稳定性与可用性

### 1.2 测试范围

| 测试类型 | 覆盖范围 |
|---------|---------|
| 功能测试 | 用户管理、座位预约、借阅管理、图书管理、OCR识别、AI推荐 |
| 接口测试 | 7个微服务 RESTful API |
| 集成测试 | 服务间 Feign 调用、RocketMQ 消息、Seata 事务 |
| 性能测试 | 预约高并发、批量查询 |
| 安全测试 | JWT 鉴权、接口防越权、SQL 注入 |

### 1.3 测试资源

| 角色 | 人员 | 职责 |
|------|------|------|
| 测试组长 | 1人 | 测试计划制定、进度跟踪 |
| 测试工程师 | 2人 | 用例编写、功能测试 |
| 开发工程师 | 4人 | Bug 修复 |

**环境要求**：
- 服务器：Docker Desktop (Windows/Linux)
- 数据库：MySQL 8.0
- 缓存：Redis 7
- 消息：RocketMQ 5.1
- 浏览器：Chrome / Edge 最新版

### 1.4 进度安排

| 阶段 | 时间 | 产出 |
|------|------|------|
| 测试计划 | 第1天 | 测试计划书 |
| 用例编写 | 第2-3天 | 测试用例库 |
| 功能测试 | 第4-6天 | Bug 清单 |
| 性能测试 | 第7天 | 性能报告 |
| 回归测试 | 第8天 | 测试总结 |

## 3. 非功能测试

### 3.1 性能测试

| 指标 | 目标值 |
|------|--------|
| 预约接口最大并发 | ≥100 QPS |
| 图书查询响应时间 | ≤500ms |
| 登录接口响应时间 | ≤200ms |
| 系统稳定运行时间 | ≥72h |
| CPU 使用率（正常负载） | ≤60% |
| 内存使用率 | ≤70% |

**工具**：JMeter 或 Postman Collection Runner

### 3.2 安全测试

| 测试项 | 方法 |
|--------|------|
| JWT Token 鉴权 | 无 token 请求应返回 401 |
| 接口越权 | 普通用户访问管理员接口应返回 403 |
| SQL 注入 | 搜索框输入 SQL 特殊字符 |
| XSS | 输入脚本标签检查过滤 |
| 验证码爆破 | 错误验证码应有限制 |
