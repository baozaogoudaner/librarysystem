# 图书馆智能服务系统 — 概要设计文档

## 1. 系统概述

### 1.1 系统定位
基于 Spring Cloud Alibaba 微服务架构的图书馆座位预约与图书管理平台，提供座位预约、图书借阅、OCR识别、AI智能推荐等功能。

### 1.2 设计目标
- **高可用**：服务注册发现、负载均衡、熔断降级
- **可扩展**：按业务边界拆分微服务，独立部署
- **数据一致**：Seata 分布式事务保证跨库一致性
- **异步解耦**：RocketMQ 消息队列削峰填谷

### 2.2 微服务划分

| 服务 | 端口 | 职责 | 数据库 |
|------|------|------|--------|
| user-service | 8081 | 用户注册登录、权限管理 | library_user |
| seat-service | 8082 | 阅览室管理、座位状态 | library_seat |
| reservation-service | 8083 | 预约CRUD、签到签退 | library_reservation |
| book-service | 8084 | 图书管理、OCR、推荐 | library_book |
| borrowing-service | 8085 | 借书还书、续借、预约 | library_borrowing |
| notification-service | 8086 | 站内通知、消息推送 | library_notification |
| gateway-service | 8080 | 统一入口、鉴权、限流 | - |

### 2.3 技术栈

| 层次 | 技术 | 用途 |
|------|------|------|
| 前端 | Vue 3 + Element Plus | 用户界面 |
| 网关 | Spring Cloud Gateway | 路由、鉴权、限流 |
| 后端 | Spring Boot 2.6 | 业务逻辑 |
| 微服务 | Spring Cloud Alibaba 2021 | 服务治理 |
| 注册配置 | Nacos 2.2 | 服务发现、配置管理 |
| 数据库 | MySQL 8.0 | 持久化存储 |
| 缓存 | Redis 7 | 缓存、分布式锁 |
| 消息 | RocketMQ 5.1 | 异步通知 |
| 事务 | Seata AT（可选） | 分布式事务 |
| 文档 | Knife4j (Swagger) | API 文档 |

## 3. 数据库设计

### 3.1 数据库拆分

| 数据库 | 包含表 | 所属服务 |
|--------|--------|---------|
| library_user | t_user | user-service |
| library_book | t_book, t_purchase_recommend | book-service |
| library_borrowing | t_borrow, t_book_reserve | borrowing-service |
| library_seat | t_room, t_seat | seat-service |
| library_reservation | t_reservation | reservation-service |
| library_notification | t_notification | notification-service |

### 3.2 核心表关系

```
t_user (用户) ──→ t_borrow (借阅) ──→ t_book (图书)
t_user ──→ t_book_reserve (图书预约) ──→ t_book
t_user ──→ t_reservation (座位预约) ──→ t_seat (座位)
t_user ──→ t_notification (通知)
t_seat ──→ t_room (阅览室)
```

## 5. 安全设计

### 5.1 认证流程
```
客户端 → 登录获取 JWT → 后续请求携带 Bearer Token
                       ↓
网关 AuthGlobalFilter → 校验 JWT → 透传 X-User-Id / X-Username
```

### 5.2 权限控制
- 白名单路径（登录、注册、验证码、图书检索）无需认证
- 管理员接口（/admin/**）需要 role=1
- 普通用户只能操作自己的数据

## 6. 部署架构

### 6.1 Docker Compose 部署
```
15 个容器：MySQL + Redis + Nacos + RocketMQ(2) + 
7个微服务 + gateway + frontend
```
- 一条命令 `docker-compose up -d` 启动全部
- 健康检查保障启动顺序
- 环境变量覆盖配置
