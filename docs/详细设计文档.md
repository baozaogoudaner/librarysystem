# 图书馆智能服务系统 — 详细设计文档

## 1. 用户管理模块（user-service）

### 1.1 类结构

```
UserController
├── getCaptcha()         → GET    /user/captcha
├── register()           → POST   /user/register
├── login()              → POST   /user/login
├── getUserInfo()        → GET    /user/info
├── updateUser()         → PUT    /user/update
├── listAllUsers()       → GET    /user/list
├── updateUserStatus()   → PUT    /user/status/{userId}
├── updateReaderPerm()   → PUT    /user/permission/{userId}
├── addViolation()       → PUT    /user/violation/{userId}    [Feign]
├── checkFreeze()        → GET    /user/check-freeze/{userId} [Feign]
└── addCreditScore()     → PUT    /user/credit/{userId}       [Feign]
```

### 1.2 关键逻辑

**登录流程**：
1. 校验验证码（Redis 中查找 captchaId → 比对 code）
2. 验证用户名密码（BCrypt 加密比对）
3. 生成 JWT Token（含 userId, username, role）
4. 返回 token + 用户信息

**注册流程**：
1. 校验验证码
2. 检查用户名唯一性
3. BCrypt 加密密码
4. 插入用户记录

### 1.3 数据表结构

**t_user**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| username | VARCHAR(50) UNIQUE | 用户名 |
| password | VARCHAR(255) | BCrypt加密 |
| real_name | VARCHAR(50) | 真实姓名 |
| role | TINYINT | 0-用户 1-管理员 |
| status | TINYINT | 0-正常 1-冻结 |
| violation_count | INT | 违规次数 |
| credit_score | INT | 信用积分 |

```
SeatController
├── listAllSeats()        → GET    /seat/list
├── listSeatsByRoom()     → GET    /seat/room/{roomId}
├── getSeatById()         → GET    /seat/{id}
├── getAvailableSeats()   → GET    /seat/available

├── markSeatReserved()    → PUT    /seat/mark-reserved    [Feign]

├── listAllRooms()        → GET    /seat/room/list

├── addSeat()             → POST   /seat/add
└── addRoom()             → POST   /seat/room/add
```

### 2.2 可用座位查询

**查询流程**：
1. 按 roomId 获取该阅览室所有座位
2. 查询该时段已有预约的 seatId 集合
3. 排除已预约的座位
4. 返回可用座位列表（含 rowNum, colNum 布局信息）

**Redis 时段锁定**：
- 预约创建前，通过 Redisson 锁定 seat:lock:{seatId}:{date}:{startTime}
- 防止高并发下重复预约

### 3.2 预约状态机

```
待签到 (0) ──[签到]──→ 使用中 (1) ──[签退]──→ 已完成 (2)
   │                      │
   ├──[取消]→ 已取消 (3)   └──[超时]→ 违规 (4)
   └──[超时未签到]→ 违规 (4)
```

**创建预约事务**：
1. 校验座位可用（Feign → seat-service）
2. 校验用户状态（Feign → user-service）
3. 本地插入预约记录
4. 标记座位已预约（Feign → seat-service，Redis锁）
5. 发送 RocketMQ 通知消息


### 4.2 推荐算法

**基于内容的推荐（Jaccard 相似度）**：
1. 获取用户借阅历史
2. 提取已借图书的 category、author 特征
3. 与在库图书计算 Jaccard 系数
4. 综合得分 = category 相似度 × 0.6 + author 相似度 × 0.4
5. 取 Top-N，排除已借图书
6. 写入 Redis 缓存（TTL 1h）

**DeepSeek AI 推荐**（可选）：
- 前端直连 DeepSeek API
- 传入借阅历史 + 馆藏列表
- 返回推荐图书 + 个性化理由

### 4.3 OCR 识别

**ZXing 条码识别**（本地）：
- 支持 EAN-13, CODE-128, QR Code 等格式
- 用于借还书扫码

**DeepSeek Vision**（可选）：
- 前端上传图片 → 调用 DeepSeek API
- 识别 ISBN/书名/作者/出版社
- 自动填充入库表单

## 5. 借阅管理模块（borrowing-service）

### 5.1 类结构

```
BorrowController
├── borrowBook()          → POST   /borrow/borrow
├── returnBook()          → PUT    /borrow/return/{borrowId}
├── renewBook()           → PUT    /borrow/renew/{borrowId}
├── getMyBorrows()        → GET    /borrow/my
├── listAllBorrows()      → GET    /borrow/list
├── getOverdueBorrows()   → GET    /borrow/overdue
├── reserveBook()         → POST   /borrow/reserve
├── cancelReserve()       → PUT    /borrow/reserve/cancel/{id}
├── getMyReserves()       → GET    /borrow/reserve/my
├── listAllReserves()     → GET    /borrow/reserve/list
└── getBorrowHistory()    → GET    /borrow/internal/history/{userId} [Feign]
```

### 5.2 借书流程

1. 检查用户是否有逾期未还 → 有则拒绝
2. 检查当前借阅数量是否达上限 → 达上限则拒绝
3. 检查是否已借此书 → 已借则拒绝
4. Feign 调用 book-service 扣减库存
5. 插入借阅记录（status=0 借出中）
6. 返回借阅详情

**还书流程**：
1. 校验借阅记录归属
2. Feign 调用 book-service 恢复库存
3. 更新借阅状态为已归还
4. 检查是否有等待中的图书预约 → 标记为可借

## 6. 通知管理模块（notification-service）

### 6.1 类结构

```
NotificationController
├── listNotifications()    → GET    /notification/list
├── getUnreadCount()       → GET    /notification/unread-count
├── markAsRead()           → PUT    /notification/read/{id}
└── markAllAsRead()        → PUT    /notification/read-all
```

### 6.2 消息类型

| type | 说明 | 触发条件 |
|------|------|---------|
| 0 | 图书到期提醒 | 定时任务检查到期借阅 |
| 1 | 逾期警告 | 定时任务检查逾期 |
| 2 | 预约可取通知 | 还书后触发 |
| 3 | 荐购进度通知 | 管理员审核荐购后 |
| 4 | 库存预警 | 库存低于阈值 |

### 6.3 RocketMQ 消费

```
reservation-service ──(出队)──→ notification-service
  Topic: reservation-topic
  消息体: ReservationMessage { userId, seatId, date, startTime, status }
  消费者: 根据 status 生成对应通知
```

## 7. 网关模块（gateway-service）

### 7.1 路由规则

| 路径 | 目标服务 | 说明 |
|------|---------|------|
| /api/user/** | user-service | 用户管理 |
| /api/seat/** | seat-service | 座位管理 |
| /api/reservation/** | reservation-service | 预约管理 |
| /api/book/** | book-service | 图书管理 |
| /api/borrow/** | borrowing-service | 借阅管理 |
| /api/notification/** | notification-service | 通知管理 |
| /api/statistics/** | reservation-service | 统计接口 |
| /api/export/** | reservation-service | 导出接口 |

### 7.2 鉴权过滤器

```java
AuthGlobalFilter (Ordered.LOWEST_PRECEDENCE - 100)
├── 白名单放行（login, register, captcha, search, book list...）
├── 校验 JWT Token → 解析 userId, username, role
├── 设置请求头 X-User-Id, X-Username, X-Role
└── 转发到下游服务
```

### 7.3 Sentinel 限流

- 网关层统一限流，防止流量冲垮后端服务
- 控制台地址：localhost:8858（可选）
